# TimFit CRM

## Что синхронизируется через Supabase

После запуска `supabase.sql` вся админка берёт данные из одной базы:

- лиды;
- клиенты и тренировки;
- профиль;
- сообщения / очередь;
- площадки;
- шаблоны;
- ручные записи CRM;
- календарные слоты для клиентской страницы.

## Обновление базы

1. Открой Supabase → SQL Editor.
2. Создай новый query.
3. Вставь **весь** файл `supabase.sql`.
4. Нажми Run.
5. После деплоя сайта **первым открой `/admin` на том устройстве, где сейчас правильные данные** — если облачная таблица состояния ещё пустая, приложение один раз перенесёт их в Supabase.
6. На втором устройстве открой `/admin` и нажми «Обновить из базы».

## Важный момент

Пока админка доступна без логина, политики Supabase открыты для `anon`. Следующий правильный этап — добавить авторизацию для `/admin`.
